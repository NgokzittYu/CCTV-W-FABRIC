# Phase 3: VIF 模块定位与边界校准完成报告

**日期**: 2026-03-28
**校准目标**: 冻结 VIF (Video Integrity Fingerprint) 的系统能力边界，确立容忍模式下的数据驱动判定准则。

## 1. 校准结论与系统定调

在真实高动态视频（如 `LE SSERAFIM` 复杂双人舞蹈实景片段）和高强度合法重编码条件下，VIF 的默认判决模式正式被重构为**宽容前置筛分模式 (Tolerant Mode)**。

该模式以**合法转码低误伤为首要目标**，在当前测试集上实现了 RE_ENCODED 样本 0 误判与 100% 保持率。
与此同时，本轮实验也明确表明，基于当前轻量视觉-时序特征的 VIF 无法独立完成同源细粒度帧替换的可靠判别。因此，**该类异常的终判职责被留给后续目标感知、细粒度校验与锚定链路处理**。

## 2. P99 上尾分布校准与阈值重定

取消过往“拍脑袋”硬编码的 `0.25` / `0.35` 阈值，全面引入基于分布探测的数据驱动配置。

通过分析 `RE_ENCODED` 集合 (H.264/H.265 最高至 CRF 35 的极度压缩) 得出的合法组特征距离分布，我们记录到：
* **最大噪声峰值 (Max Risk)**: `0.3321`
* **第 99 百分位点 (P99 Risk)**: `0.3203`

针对上述分布，我们将 `tri_state_verifier.py` 的架构改版为解耦配置：
* `tolerant_threshold = 0.35`: 作为默认主线，基于合法转码分位上尾 (P99 + margin) 确立。
* `ablation_threshold`: 作为接口预留，以供后续或消融实验临时载入，彻底避免将特定数据的“基准补偿值”神化为通用环境下的真理。

## 3. 基准测试快照 (Benchmark Snapshot v1.4.1)

**当前固定系统权重**:
- $W_{vis} = 0.50$
- $W_{tem} = 0.50$
- $Threshold = 0.35$

**测试集落盘组成**:
- INTACT: 1
- RE_ENCODED: 30
- TAMPERED: 7 (同源微小替换 1~15帧)

**四项核心基准指标**:
1. **Macro-F1 Score**: 0.6318
2. **合法转码保持率 (RE_ENCODED Retention Rate)**: 30/30 (100.0%)
3. **合法转码误判率 (RE_ENCODED False Positive R)**: 0/30 (0.0%)
4. **细粒度篡改漏判率 (TAMP False Negative Rate)**: 7/7 (100.0%)

总结：VIF 模块作为 SecureLens 系统防线的第一道前置大闸，基于以上快照正式“钉死边界”。从此它只负责处理能够突破重编码底噪极限的粗粒度或大面积破坏。至此，Phase 3 的 VIF 结构探索收口完毕。
